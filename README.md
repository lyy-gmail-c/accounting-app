# MiniBook · 极简记账

> 一款面向个人的极简记账应用 —— **一个网址，双端同时可用**：手机端便捷记账，电脑端科学对账，数据本地存储，离线可用。

- 🌐 **在线访问**：[https://lyy-gmail-c.github.io/accounting-app/](https://lyy-gmail-c.github.io/accounting-app/)
- 📦 **项目地址**：[https://github.com/lyy-gmail-c/accounting-app](https://github.com/lyy-gmail-c/accounting-app)

---

## ✨ 核心特点

### 📱💻 同一个网址，双端不同形态

**同一 URL 在手机和电脑上打开会自动切换到完全不同的界面** —— 无需下载 App、无需分别部署、无需切换账号：

| 端 | 触发条件 | 定位 | 典型场景 |
|----|----------|------|----------|
| **手机端** | 屏幕宽度 < 768px | 数据采集器 | 随手记一笔支出/收入，3 步搞定 |
| **电脑端** | 屏幕宽度 ≥ 768px | 数据管理中心 | 月末对账、分类统计、账户结余核算 |

> 应用通过 CSS 媒体查询自动判断设备形态并渲染对应 UI，两端共享同一套 IndexedDB 数据。

### 🔒 数据本地存储，隐私安全
- 数据完全保存在浏览器的 **IndexedDB** 中，**不上传任何云端服务器**
- 无账号体系、无网络同步、无第三方分析 —— 你的账本只属于你
- 需要跨设备同步：手机端导出 JSON → 电脑端导入（规则引擎智能补齐），数据流完全在本地掌控

### 🌐 PWA 离线可用
- 首次访问后自动缓存到本地，**无需网络也能正常使用**
- 支持添加到主屏幕，启动体验接近原生 App
- Service Worker 缓存策略确保加载迅速

### 🎨 极简设计
- 零外部依赖：**纯 HTML + 原生 JS + CSS**，无任何第三方库
- 启动即用，页面轻量，体积不到 200KB
- 中式财务惯例：涨红跌绿、¥ 人民币符号

## 🚀 快速开始

### 线上使用
直接打开：[https://lyy-gmail-c.github.io/accounting-app/](https://lyy-gmail-c.github.io/accounting-app/)

## 📱 手机端功能 —— 便捷记账

**给你一部手机，随时记一笔。不打断生活，不增加心智负担。**

### 📝 记一笔，只要 3 秒
- 打开即记账，无需登录、无需选账户
- 只分 **支出 / 收入** 两类，大按钮一点就切
- 分类极简：**衣 / 食 / 住 / 行 / 其他**，再没有"我这笔算哪一类"的纠结
- 金额 → 分类 → 保存，记完即走

### 📒 查账清爽不累
- 明细页按月展示，**往下划就自动加载上个月**，不用翻页不用选年月
- 月份标题始终吸顶，滑再远也知道看的是哪个月
- 看到错账 **一划就删**，点一下就改

### 📊 一眼看清收支
- **按月列出** 这个月花了多少、赚了多少、结余多少，一整年全在那儿
- 首屏显示 **总余额**，心里有数
- 想看去年同期？直接划下去就行

### 📤 一键分享给电脑
- 一键导出账单，可以直接通过 **微信 / 邮件 / AirDrop** 发到电脑端
- 电脑端接收后会自动补齐分类和账户，手机端记得轻松，电脑端算得准确

## 💻 电脑端功能 —— 科学对账

**月底算账、对单、做预算，把每一分钱的去向都看清楚。**

### 📝 精细化记账
- 支持 **支出 / 收入 / 转账** 三种类型，转账会自动带"从哪里转到哪里"
- **多账户管理**：微信、支付宝、银行卡、现金，随便加
- **三级分类**：从大类到小项，账目想多细就多细（默认已内置 46 个分类够用）

### 🔍 想怎么查就怎么查
- 明细表格支持 **按时间、账户、分类、类型** 任意组合筛选
- 一键导出 **CSV**，拿去 Excel 里做二次分析也行
- 每一笔都可以 **编辑 / 删除 / 复制**，改账不怕麻烦

### 📈 看得见的收支趋势
- 本周 / 本月 / 本年 / 自定义区间，**一切随时间随你选**
- 收入、支出、结余 **折线图**，趋势一目了然
- **按账户统计** —— 对银行卡、对微信余额的神器
- **按分类统计** —— 看看钱到底花在哪了，占比多少

### 🗂️ 分类和账户，随你组织
- 分类、账户都可以 **自由增删改**
- **拖拽排序**，常用的放上面，顺手
- 每个账户支持设 **初始余额** 和 **备注**，老账迁入毫无压力
- 余额自动算，不用你手动加减

### 💾 数据完全掌控
- **一键全局备份**：一份文件打包所有记录、分类、账户、设置
- **全局恢复**：换电脑、重装系统？导入一下就回来了
- **按范围导出**：只导出 3 月的账单？只导出微信的流水？都行
- **同步手机端账单**：手机发来的流水会智能预填分类和账户，你确认一下就进库
- **分类 / 账户配置可单独导入导出**，方便在多台设备上保持一致

## 🛠 技术栈

| 模块 | 技术 |
|------|------|
| 前端 | 原生 HTML + JS + CSS（零第三方库） |
| 数据存储 | IndexedDB（浏览器本地，4 表：records / categories / accounts / settings） |
| 响应式分流 | CSS 媒体查询（768px 分界） |
| 离线能力 | Service Worker + Cache API |
| 图标 | 内联 SVG（Lucide 风格） |
| 部署 | GitHub Pages（静态托管） |

## 📂 项目结构

```
├── index.html          # 单页入口，双端布局容器
├── manifest.json       # PWA 配置
├── sw.js               # Service Worker（Cache First）
├── css/
│   ├── base.css        # CSS 变量 + Reset + 通用组件
│   ├── mobile.css      # 手机端样式 @media <768px
│   └── desktop.css     # 电脑端样式 @media ≥768px
├── js/
│   ├── app.js          # 应用入口 + SVG 图标系统 + 时间工具
│   ├── db.js           # IndexedDB 数据层（CRUD + 统计 + 导入导出）
│   ├── mobile.js       # 手机端 UI（记账 / 明细 / 统计 / 我的）
│   └── desktop.js      # 电脑端 UI（明细 / 统计 / 分类 / 账户 / 数据管理）
└── icons/              # PWA 图标
```
